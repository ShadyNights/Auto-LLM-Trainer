# Database Schema

> [!IMPORTANT]
> PostgreSQL serves as the definitive single source of truth for the entire platform. The schema is entirely unified into `migrations/001_initial_schema.sql` and prohibits business logic (triggers) from leaking into the data layer.

---

## 1. Core Domain Entities

These tables represent the fundamental operational data generated by user interactions.

- **`users` & `conversations`**: Manages unique user identity and distinct session tracking.
- **`events`**: A strictly-typed, immutable ledger of user interactions (`event_type`), linked via `correlation_id`. Uses `JSONB` for extensible telemetry payload storage.
- **`itineraries`**: The concrete output. Contains the generated travel plans alongside a frozen `configuration_snapshot` (capturing hyperparameters like temperature).

## 2. Telemetry & Learning Queue

These tables orchestrate the asynchronous Continuous Feedback Learning Pipeline.

- **`training_queue`**: The active background queue. Tracks task state, retries, and durations. Unresolvable tasks automatically transition to a Dead Letter Queue (DLQ).
- **`datasets`**: Represents curated training data built asynchronously by the pipeline. Supports lineage tracking via `parent_dataset_id`.

## 3. Configuration & Registry

These tables govern the operational parameters and versioning of the LLM systems.

- **`system_config`**: A singleton table managing global toggles, feature flags (e.g., `learning_enabled`), and active model routing.
- **`prompts_metadata`**: Manages prompt versioning, filesystem checksums, and deployment status.
- **`model_versions`**: The central registry tracking combinations of Providers, Models, Prompts, and Datasets.
- **`config_evaluations`**: Stores quantitative benchmarks (`structural_metrics`, `semantic_metrics`) for respective configurations.

## 4. Helper Functions

Pure, side-effect-free SQL functions to assist the background pipeline:
- `calculate_quality_score()`: Normalizes 0-5 user ratings into a standardized dataset score.
- `extract_training_sample()`: Formats relational itineraries into LLM-ready `JSONB` training dictionaries.
